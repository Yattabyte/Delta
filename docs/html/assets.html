Engine Assets

Engine assets are an encapsulation of a file or set of related files that are found within the application subdirectory.
Different file types need to be handled in different ways. Assets are designed to be ultra-efficient.
We sort and store them within an asset manager, and we share them whenever possible to avoid redundant disk reads.
Lastly, all assets supported thus far support multithreading, as to avoid locking the main thread.
Here's a list of all asset types the engine currently supports.

  • Collider
  • Config
  • Cubemap
  • Image
  • Level
  • Material
  • Mesh
  • Model
  • Primitive
  • Shader
  • Shader_Geometry
  • Shader_Pkg
  • Texture

New Assets

All assets have provide the following functionality - a static Create(...) function, and an overridden virtual initialize(...) function:

  • void Shader::initialize( Engine * engine, const std::string & relativePath )
